# v0.5.0 dependency benchmark current-state audit

Audit scope: tracked repository surfaces only; no network or local data cache
was accessed. This is an architecture inventory, not biological validation.

## Existing benchmark surface

`targetintel.benchmark`, `scripts/08_run_benchmark.py`, and
`configs/benchmark_targets.yaml` implement the legacy 56-target internal
therapeutic-intent sanity benchmark. It accepts a ranked TargetIntel table,
merges expected stable roles and primary/acceptable intents, and writes
per-target predictions, role confusion, intent metrics, and global coverage,
role accuracy, intent accuracy, MRR, and top-K metrics. Missing targets are
retained with a missing-prediction sentinel. Roles and primary intents come
from the legacy benchmark configuration. It has no development/holdout
partitioning, frozen-universe identity, resistance-axis reporting, source
ablation, profile reconciliation, essentiality stratification, or diagnostic
rank-stability comparison.

The legacy positive examples are therapeutic-intent expectations; its controls
include context and poor-direct-target groups. These labels are appropriate for
the legacy sanity check but must not be joined into a dependency signal.
Current rank/score fields are the unchanged TargetIntel ranked-table rank and
intent scores. The legacy evaluator does not modify them, but it is not the
canonical Issue 505 implementation because it would couple this work to
production-ranking schema and labels.

## Selected canonical surfaces and gaps

Issue 504's `targetintel.functional_dependency.target_universes` provides
frozen benchmark, discovery and background TSVs plus an immutable freeze
manifest. Benchmark rows retain controlled classes, partition, and resistance
axes. Issue 503's `depmap_profiles` provides immutable profile-run identity,
continuous contrasts, empirical lineage position, coverage states, and separate
common-essential/pan-dependency reference states. Issue 505 therefore adds
`targetintel.functional_dependency.depmap_benchmark` as the sole analysis-only
reconciliation and diagnostic-ranking surface. It reads a separate explicit
baseline artifact and does not reuse or duplicate production ranking.

Before Issue 505 there was no reproducible bridge between frozen identities,
profile runs and baseline rank artifacts; no controlled class map; no separate
development/holdout metrics; no dependency-signal definition; no bounded local
overlay; no ablation or rank-stability layer; and no enrichment or integration
evidence record. These gaps prevent a reproducible assessment of DepMap
evidence. The synthetic fixture remains operational only and cannot establish
melanoma performance.

# v0.5.0 DepMap current-state audit

Audit date: 2026-07-19. Scope: tracked source, tests, scripts, configuration,
examples, and documentation. No network, portal, API, or local-data-cache access
was performed.

## Findings

There is no current DepMap implementation. A repository-wide case-insensitive
scan for DepMap, CRISPR, Chronos, Achilles, gene effect, dependency probability,
common essential, pan-essential, ModelID, and lineage found only roadmap or
release-boundary prose, plus unrelated biological use of lineage. There are no
DepMap modules, clients, downloaders, matrix parsers, metadata mappers,
dependency profiles, cached data, fixture datasets, or DepMap-specific tests.

| Area | Current state |
| --- | --- |
| Inputs and cached/fixture data | None: no real, cached, fixture, or synthetic DepMap input. |
| Derived fields | None. Candidate `depmap_*` fields occur only as future-roadmap text in `docs/ROADMAP_2_0.md` and documentation regression assertions. |
| Gene identity | No DepMap column-label, approved-symbol, Entrez, alias, or malformed-label handling. |
| Model identity / melanoma selection | No model metadata mapping, `ModelID` handling, row matching, or melanoma selector. |
| Release and checksums | No release identifier, source manifest, schema fingerprint, checksum, or retrieval identity. |
| Missingness | No dependency missingness model; data absence and no dependency are not represented in this domain. |
| Scoring and ranking | No DepMap data affect features, roles, scores, rankings, benchmark results, cards, or reports. |
| Operational scripts | Numbered feature/scoring/card/report scripts contain no DepMap work. |
| Existing coverage | `tests/test_release_documentation.py` only guards roadmap wording; pipeline, scoring, feature, role, modality, Open Targets, LLM, card, and report tests contain no DepMap integration. |

The complete list of tracked DepMap-specific references before this issue is:
`README.md`, `CHANGELOG.md`, `docs/releases/v0.2.0.md`,
`docs/ROADMAP_2_0.md`, `docs/specs/v0.2.0_evidence_layer.md`,
`docs/specs/v0.3.0_grounded_literature_copilot.md`, and
`tests/test_release_documentation.py`. They describe future work only.

## Architecture decision

No reusable operational surface exists. Issue 501 therefore designates
`targetintel.functional_dependency` as the sole future domain-contract surface.
`depmap_models.py` owns immutable release/file/schema/layout contracts and
`depmap_validation.py` owns local header-level validation. Issue 502 must extend
this package rather than create a downloader, matrix parser, model mapper, or
competing dependency-profile surface. No legacy code exists to retire, and this
issue does not replace behavior.

## Gaps blocking reproducible real-data execution

No pinned real release, checksums, real headers, release README, model metadata,
CRISPR files, melanoma-context mapping, gene-label mapping, target universe,
derived outputs, coverage, or statistical implementation exists. These gaps
block real-data or biological conclusions until later release-gated issues.

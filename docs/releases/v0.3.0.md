# TargetIntel-IO v0.3.0 — Grounded evidence and synthesis infrastructure

## Release purpose

v0.3.0 closes a research-infrastructure cycle: it composes provider-neutral LLM execution with source-grounded extraction, deterministic scientific claim audit, mandatory human review, controlled `EvidenceItem` promotion, explicit persistence, reviewed-evidence snapshots, grounded target synthesis, and safe Obsidian Markdown export.

The included [offline mock demo](../../examples/llm/README.md) exercises that complete chain using fabricated `DEMO_*` content and `MockProvider` only.

## Architecture and reproducibility

Provider execution is neutral to a model backend. Extraction candidates retain source quotation and identity; audit, review packet, review decision, promotion, persistence receipt, snapshot, synthesis, export plan, export receipt, and note-content identities are deterministic. Operational timestamps and local paths are kept outside scientific identities. Persistence is explicit and snapshots are read-only views of reviewed records. Synthesis is limited to a selected target and its reviewed snapshot EvidenceItem IDs; it preserves contradictory, null, limiting, and uncertain evidence when supplied.

Human review is mandatory before promotion. Approval means permission for controlled promotion within the software workflow, not clinical or scientific validation. Obsidian export is explicit, containment-checked, content-hashed, and idempotent for identical content; Obsidian is not a scientific source of truth.

## Security and boundaries

The contracts reject hidden-reasoning and secret-bearing fields. The mock demo requires no network, API key, server, model download, GPU, database service, or Obsidian installation. It writes only under the caller-selected output directory.

v0.3.0 does not validate targets or biomarkers; establish causality or clinical utility; provide treatment recommendations; replace expert scientific review, experimental validation, or clinical validation; or prove LLM-generated text scientifically true. It does not rank targets through the LLM layer and does not change the deterministic scoring pipeline. It adds no production CLI command, provider, retrieval, scoring, ranking, role classifier, or evidence semantics.

## Known limitations and deferred work

The demo is synthetic and demonstrates software contracts, not biological findings. LLM output remains a structured, audited research artifact requiring human review and independent validation. Production-provider operations, additional data sources, and any clinical or experimental validation remain deferred.

## Migration notes

No migration is required. The deterministic pipeline remains unchanged. v0.3.0 APIs are opt-in, explicit, and separate from deterministic scoring and ranking.

# v0.5.0 — DepMap/CRISPR functional dependency

TargetIntel-IO v0.5.0 adds a deterministic, local-only functional-dependency
layer based on DepMap CRISPR gene-effect and dependency-probability data.

The module is explanatory and post-ranking. It characterizes whether targets
show dependency patterns consistent with the selected melanoma context, but it
does not automatically replace the existing therapeutic-intent ranking.

## Implemented workflow

The release-closure workflow includes:

- immutable DepMap release manifests and schema fingerprints;
- checksum verification before ingestion;
- full-release and requested-target ingestion;
- benchmark, discovery, and genome-wide background universes;
- explicit target-identity reconciliation;
- controlled resistance-axis annotations;
- reviewed melanoma model-context definitions;
- gene-effect and dependency-probability profiles;
- benchmark coverage, enrichment, control, ablation, and stability analyses;
- bounded dependency-aware ranking overlays;
- baseline-preservation and candidate-activation gates;
- internal replica comparison and external run-to-run reproducibility checks.

## Real-data execution

The complete workflow was executed using `DepMap_Public_26Q1`.

The primary context contains 56 reviewed cutaneous melanoma models. Four acral
melanoma models are retained as a separate sensitivity cohort. Drug-adapted
derivatives are excluded from the primary context.

The frozen universes contain:

| Universe | Targets |
|---|---:|
| Curated benchmark | 56 |
| Discovery after required benchmark union | 331 |
| DepMap background | 18,531 |

Benchmark and holdout coverage were both 100%.

All six required stages completed:

1. full-release ingestion;
2. requested-target ingestion;
3. target-universe freezing;
4. dependency profiling;
5. benchmark evaluation;
6. bounded integration.

The resulting state is
`ready_research_preview_human_review`.

## Scientific boundary

This release establishes technical execution and reproducibility for the local
public-data workflow. It does not establish clinical validation, causal target
validation, anti-PD-1 treatment-response prediction, or authorization to
advance a therapeutic candidate.

DepMap cell-line viability dependencies are treated as context and feasibility
evidence rather than direct evidence of clinical efficacy.

The source matrices, local run directories, and machine-specific configuration
paths are intentionally excluded from version control.

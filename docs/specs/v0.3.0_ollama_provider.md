# v0.3.0 Ollama provider adapter

## Boundary

`OllamaProvider` adapts immutable, provider-neutral `LLMRequest` values to one
non-streaming local Ollama `/api/chat` operation and normalizes the result into
an immutable `LLMResponse`. It is operational model output only: it does not
retrieve, construct `EvidenceItem` values, validate citations, store data, or
alter deterministic classification, ranking, or scores.

## Request mapping

The configured model, `stream: false`, ordered system and user messages, and
an optional configured `keep_alive` are sent as canonical JSON. The system
instruction is the first `system` message. The user message contains the user
instruction followed by a labelled `<source_document>` envelope whose ID is
the request source-document ID. Text is preserved exactly; structured source
content is serialized with the contract's canonical JSON.

Only the neutral generation settings `temperature`, `seed`, `top_p`, and
`num_predict` are accepted and are placed in Ollama `options`. Unknown keys
fail before transport execution. The adapter's deterministic default is an
empty options object; callers can explicitly set a supported deterministic
seed or temperature. No request metadata, arbitrary payload properties,
headers, credentials, or environment variables are forwarded.

If a schema resolver is supplied, it must resolve the exact `(schema ID,
schema version)` pair before transport execution. The resolved JSON Schema is
sent unchanged through Ollama's `format` field. A missing schema is an invalid
local request, not a network attempt.

## Transport and offline development

The provider accepts an injected `HTTPTransport`, whose `send(method, url,
body, headers, timeout)` returns `HTTPResponse(status_code, headers, body)`.
Tests use a fake transport to simulate success, malformed bodies, HTTP errors,
timeouts, and connection failures with no network, Ollama installation, model
weights, GPU, or credentials. `UrllibTransport` is a standard-library option
for a future explicit local smoke test; it has no retries.

Configuration defaults to `http://127.0.0.1:11434`, requires a non-empty model
and finite positive timeout, and rejects URL userinfo, query strings, and
fragments. Local operation has no authentication support: cloud Ollama and
API-key authentication are out of scope. Configuration serialization has no
credential fields.

## Responses, capabilities, and errors

Successful replies require a complete `done: true` response and assistant
`message.content`. `thinking` and other hidden reasoning are ignored. When a
schema is configured, raw assistant content is retained and parsed only as a
JSON object; invalid JSON or non-object JSON is `malformed_output`, never an
empty success. `total_duration` is Ollama nanoseconds and is converted to
milliseconds by dividing by 1,000,000. `prompt_eval_count` and `eval_count`
become `prompt_tokens` and `generated_tokens`. `created_at` is operational
metadata, as are timing values, and neither changes deterministic response
identity.

The response preserves the returned model name and separately records the
configured requested model name for mismatch auditing. Model-version reporting
and maximum context size remain unknown. The adapter declares system
instructions, JSON Schema/structured output, token usage, deterministic seed,
and local execution support.

HTTP 400 is invalid request, 404 is not configured/missing model, 429 and 5xx
are retryable, and unknown statuses are permanent failures. Empty or malformed
HTTP JSON, provider error objects, missing messages/content, incomplete
responses, and malformed structured output remain visible through the neutral
status/error taxonomy. Audit-facing messages are fixed, short, and do not copy
HTTP bodies, headers, URLs, exceptions, or secrets.

## Limitations

This adapter does not install, download, start, stop, or invoke Ollama. A
future optional real-local smoke test must be opt-in. Prompt-injection defence,
full schema validation, tool calls, images, streaming, grounded scientific
extraction, citation verification, evidence finalization, and target-level
synthesis are intentionally deferred.

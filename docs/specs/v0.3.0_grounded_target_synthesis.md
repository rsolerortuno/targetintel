# Grounded target synthesis v0.3.0

Issue 309 defines a research-only communication layer between a validated `ReviewedEvidenceSnapshot` and a future explicit export layer. It accepts only an immutable snapshot and an explicit, deterministic target request; it neither opens an evidence store nor retrieves, persists, scores, ranks, exports, or writes files.

`TargetSynthesisRequest` uses controlled purpose and section vocabularies, exact `EvidenceItem.target_symbol` matching, and optional exact `EvidenceItem.disease_name` context matching. Request identities exclude operational timestamps. A deterministic inventory retains selected item IDs, payload hashes, observations and interpretations separately, structured limitations/context/provenance, and opposing or null directions without choosing a winner.

The versioned prompt labels inventory values as untrusted data and requires strict JSON. It prohibits embedded prompt instructions, external knowledge, invented identifiers or quantities, hidden reasoning, clinical guidance, recommendations, scoring, and ranking. Provider calls use the existing provider-neutral `LLMRequest`/`LLMResponse` and single-call execution service.

Every generated statement cites snapshot item IDs. Reference/identity, complete-coverage, lexical safety, and boundary validation are deterministic; citation attachment does not establish full semantic entailment. Existing Issue 305 therapeutic and clinical lexical taxonomy is reused. Contradictions, limitations, uncertainty, and unsynthesized items remain visible.

The immutable result preserves request, snapshot, inventory, prompt, provider request/response, and statement identities, allowing the Issue 303–308 provenance chain to continue through synthesis. The pure Markdown renderer returns text only, includes deterministic evidence markers, and states research-only/non-clinical/no-score boundaries. Operational CLI, file/Obsidian export, and persistence are deferred.

# v0.5.0 dependency benchmark

Issue 505 is an offline, analysis-only assessment layer. It consumes a frozen
Issue 504 universe, an Issue 503 profile run, an explicit unchanged baseline
ranking, and an immutable evaluation policy. It makes no clinical, causal,
safety, therapeutic-success, or production-ranking claim.

## Contracts and reconciliation

`DependencyBenchmarkPolicy` is versioned and content-addressed. Its sorted
payload fixes partitions, class mapping, K values, coverage/eligibility
thresholds, enrichment convention, candidate definitions, predeclared
ablations, stability thresholds, missing-profile and tie policies. Unknown
fields, hidden reasoning, credentials, target-specific settings, and dynamic
callbacks are rejected. Policy order cannot change identity.

The baseline TSV requires original identifier, canonical identity, contiguous
positive baseline rank, primary intent and role; score and selected state are
optional. Duplicate identities, invalid ranks and changed input fingerprints
fail closed. The evaluator retains every benchmark entry explicitly as ranked
and profiled, ranked not profiled, profiled not ranked, or unresolved, and also
records discovery/background absence. No inner join is used.

Frozen benchmark class mapping is policy-declared: positives and negatives are
eligible controls, while challenging, mechanism and biomarker-only classes are
visible descriptive controls rather than automatic therapeutic positives.
Development, holdout and combined-descriptive metrics are emitted separately.

## Diagnostic rankings and metrics

The dependency-only signal uses only policy-declared profile components:
negative context-minus-non-context gene-effect contrast (reversed so more
negative is stronger), positive dependency-probability contrast, and empirical
lineage position. Each is transformed to a deterministic within-universe
mid-rank percentile and equally averaged when the declared component minimum
is met. Missingness remains explicit. Essentiality references are descriptive
strata, never hidden penalties.

The dependency-only order ties by baseline rank then canonical identity. The
bounded overlay has a fixed policy band size; it only reorders profiled,
eligible targets within their baseline band, retaining ineligible order. Its
band-violation metric must be zero. Both ranks are labelled analysis only and
retain baseline rank/score beside them.

Metrics retain denominators: coverage, Recall@K, Precision@K, average
precision, reciprocal-rank summary, median and normalized median rank,
positive/negative top-K counts and negative exclusion. Candidate comparisons
include Spearman rank correlation, top-K overlap/churn, absolute rank change,
and overlay band violations. Axis output handles multi-axis entries as members
of each listed axis and marks small axes insufficient. Exact enrichment is
exploratory; invalid small configurations are reported unavailable and never
used to tune the policy.

## Outputs and Issue 506 handoff

`examples/depmap/run_dependency_benchmark.py` writes canonical JSON and stable
TSVs including coverage, baseline/candidate metrics, rank/control/partition/
axis/ablation reports, exploratory enrichment, integration evidence, Markdown,
and a terminal manifest. The integration result always remains
`human_review_required`; it cannot register a production profile. The bundled
fixture is synthetic and does not demonstrate real melanoma performance. A
real local frozen release and human review are required before Issue 506 may
consider an optional profile.
